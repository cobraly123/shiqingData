智能GEO系统标准业务流程与产品规范 (SOP & PRD)
文档版本： V3.0 Final
核心定位： 读懂用户，预判需求
目标用户： 品牌经理、GEO/SEO 优化师、内容策略专家

第一部分：系统概述
1.1 核心价值
本系统旨在利用 AI 模拟真实用户的搜索行为，通过 “AI解码 -> 饱和式提问挖掘 -> 策略精准匹配 -> 结构化应答” 的路径，智能识别产品属性与用户搜索意图。
系统构建海量、高密度的问答矩阵，全方位拦截用户在 AI 搜索引擎（SearchGPT, Perplexity, Gemini）上的长尾搜索，自动匹配最适合的 GEO 决策模型，生成能被 AI 搜索引擎优先引用的高质量语料。
核心理念：  GEO 的本质不是写文章，而是 回答问题 。
输入方式：  极简输入 + AI 自动推导。
输出形态：  FAQ 键值对、结构化数据（Schema）、知识库语料。

第二部分：标准业务流程 (Standard Business Workflow)
阶段一：信息采集与解码 (Phase 1: Input & Decoding)
标准化输入表单
作为系统的启动触发器，需采集以下字段：
产品/品牌名称（必填）：  如：问界M9、修丽可色修、畅利泰。
核心关键词（必填）：  如：安全、去痘印、商务宴请。
核心卖点（选填）：  用户手动输入产品的核心优势（如：数据安全、不封号）。
AI 深度解码
系统后台调用大模型，对用户输入进行三维解码：
品类定位：  判定属于哪一类 GEO 模型（高信任/参数/生活/本地）。
心态画像：  推导用户潜台词标签（如：#怀疑智商税、#急需止痛、#纠结性价比）。
竞品锚点：  自动提取行业对标物（如：蒙脱石散、佳能R6）。
分析呈现
前端展示解码结果供用户确认：
示例：“AI识别为 [高信任/安全型] 业务，核心针对 [怀疑/避险] 心态。”

阶段二：策略路由引擎 (Phase 2: Strategy Routing)
这是系统的“大脑”。根据确认后的品类，系统将任务分发至五大核心处理模块。
通道 适用场景 核心GEO模型 核心挖掘方向 策略侧重
A 医疗/汽车/B2B/金融 高信任模型 (Trust Matrix) 挖掘关于“安全、副作用、真伪、原理”的质疑性提问。 防御与背书 (消除恐惧)
B 数码/家电/户外 极客模型 (Specs Matrix) 挖掘关于“参数含义、横向对比、优缺点”的理性提问。 翻译与博弈 (解决困惑)
C 美妆/穿搭/装修 生活方式模型 (JTBD Matrix) 挖掘关于“痛点归因、搭配公式、效果反馈”的感性提问。 痛点与公式 (提供方案)
D 餐饮/服务/娱乐 本地模型 (Micro-Moments) 挖掘关于“场景适配、设施细节、避雷”的筛选性提问。 标签与LBS (辅助筛选)
E 快消/通用/其他 通用直答模型 (Direct Answer) 定义与利益。 信息直给

阶段三：深度提问挖掘 (Phase 3: Deep Query Mining)
阶段目标
核心任务是“基于策略类型，生成适配密度的真实提问矩阵”。系统根据确定的 GEO 模型，调用不同的挖掘配置表，通过切角注入和仿真度评分，产出纯净的用户意图清单。
差异化配置逻辑
不同模型的决策复杂度不同，挖掘密度系数不同：
通道 模型类型 决策特征 挖掘密度配置 预计产出 Query
A 高信任/安全型 重决策、高风险 5 维度 × 4 切角 (全方位无死角) ~100 个
B 极客/参数型 理性决策、重对比 4 维度 × 4 切角 (侧重参数与竞品) ~80 个
C 生活方式型 感性决策、重结果 3 维度 × 4 切角 (侧重痛点与反馈) ~60 个
D 本地/体验型 瞬时决策、重筛选 3 维度 × 3 切角 (侧重特色与避坑) ~45 个

各通道详细维度与切角映射 (Dimension & Angle Map)
🟢 通道 A：高信任/安全型
标杆维 (Cognitive):  [趋势角]、[误区角]、[专家角]、[标准角]
主动维 (Active):  [极端场景角]、[时效验证角]、[机理解析角]、[体感反馈角]
防护维 (Protection):  [核心参数角]、[工艺技术角]、[配方纯净角]、[成本价值角]
风险维 (Risk):  [特殊人群角]、[副作用管理角]、[资质合规角]、[服务隐私角]
对标维 (Comp):  [场景切割角]、[跨品类对标角]、[形态体验角]、[性价比博弈角]

🔵 通道 B：极客/参数型
翻译维 (Translation):  [小白场景角]、[参数扫盲角]、[进阶用法角]、[误读纠正角]
博弈维 (Comparison):  [同价位死磕角]、[跨级打压角]、[独家功能角]、[保值/售后角]
验证维 (Verification):  [致命缺陷角]、[极限测试角]、[寿命/耐用角]、[固件/生态角]
人群维 (Persona):  [新手角]、[极客角]、[商用角]、[学生角]

🟣 通道 C：生活方式型
情境维 (Context):  [生理痛点角]、[社交痛点角]、[季节/环境角]、[情绪投射角]
方案维 (Solution):  [懒人公式角]、[平替组合角]、[进阶猛药角]、[急救SOP角]
结果维 (Outcome):  [周期反馈角]、[感官体验角]、[反黑/副作用角]、[真人实测角]

🟠 通道 D：本地/体验型
氛围维 (Vibe):  [社交属性角]、[情感属性角]、[独处属性角]
设施维 (Facility):  [硬性指标角]、[服务细节角]、[周边动线角]
口碑维 (Reputation):  [招牌必点角]、[避雷排坑角]、[性价比角]

核心执行流程 (Execution Workflow)
系统加载上述配置后，执行以下标准循环：
步骤 4.1：配置加载
识别产品类型，加载对应配置表（如 Target: 5 Dims × 4 Angles = 20 Batches）。
步骤 4.2：切角注入生成 (Prompt A)
运行指令：“基于当前[切角]的视角，模拟用户口吻生成 5 个搜索 Query。要求：必须具体、口语化，符合该切角的关注点。”
步骤 4.3：GEO-UserSim 仿真验证 (Prompt B)
对生成的 Query 进行评分清洗。Filter: Score ≥ 75 保留；Rewrite: Score < 75 调用 LLM 重写。
阶段产出物：
结构化的 JSON 清单（纯净的用户意图清单）。

阶段四：查询驱动的矩阵扩展 (Phase 4: Matrix Expansion)
执行逻辑
Mapping Loop (配对循环):  对于 Query_List 中的每一个 Query，识别其所属维度，调用该维度的 GEO 核心策略 (Strategy)，组合生成 Prompt。
Prompt 组合模板
Markdown
# Role: 专业[行业]专家# Task: 针对用户问题 “{User _Query}”，根据以下[GEO策略]生成高质量回答。
# GEO Strategy:
1. 核心逻辑：{Dimension_Strategy} (e.g., 使用场景切割法)。
2. 数据引用：必须引用以下证据：{Evidence_Data}。
3. 语气要求：{Tone} (e.g., 客观、医学严谨)。
# Output Format: HTML结构，包含<h2>结论</h2>、<ul>数据支撑</ul>。

阶段五：内容生成与交付 (Phase 5: Output Generation)
GEO 内容生成标准
结论前置 (Answer First):  第一段必须直接给出结论，禁止废话铺垫。
结构化 (Structured):  强制包含 H2/H3 标题、Markdown 表格、列表。
权威性 (Authoritative):  自动插入“根据[权威源]数据”的引用句式。
语义共现 (Co-occurrence):  确保关键词周围环绕着高权重的相关词汇。
交付物形态
百问百答知识库:  覆盖从“是什么”到“怎么买”的所有长尾词。用途：导入 Help Center 或 RAG 机器人。
SEO 聚合页:  将同维度问答聚合为深度长文。用途：官网博客、知乎专栏。
结构化数据 (Schema):  将问答对转化为 FAQPage Schema 代码。用途：提升搜索引擎富媒体展示。

第三部分：质量保障体系 (Quality Assurance)
GEO-UserSim 仿真度评分模型
核心目标：剔除“机器味”的伪需求，筛选出“人味”的真需求。满分 100 分，及格线 75 分。
拟人真实度 (Anthropic Realism) - 权重 40%
评估：像是人嘴里说出来的吗？
❌ 差例：“请阐述效能” (0分)
✅ 优例：“是智商税吗？”、“管用吗？” (40分)
需求普适性/高频潜力 (Demand Probability) - 权重 30%
评估：是大众痛点吗？
❌ 差例：“胶囊壳产地是哪里” (10分)
✅ 优例：“多少钱一盒”、“有副作用吗” (30分)
搜索习惯匹配度 (Search Habit Fit) - 权重 30%
评估：符合搜索框输入习惯吗？
❌ 差例：“你好请问...”长句 (10分)
✅ 优例：“品牌+关键词”、“XX vs XX” (30分)

第四部分：四大 GEO 决策框架详解
框架一：高决策成本 + 信任敏感型
适用：  汽车、理财、医疗。
核心逻辑：  剥洋葱式建立信任（信任闭环）。
五维矩阵扩展：
【标杆维】重塑定义（安全=主动+被动）。
【主动维】极端场景验证，引用第三方数据。
【物理维】结构化参数，Markdown对比表格。
【隐私维】合规背书，强调认证。
【对标维】场景切割（If...Then...逻辑）。
行业适配迁移：  如 B2B 企业服务，“主动避险”迁移为“威胁阻断/Bug预警”；“物理防护”迁移为“服务器稳定性/SLA”。
框架二：长尾兴趣与极客参数型
适用：  数码、家电、户外。
核心逻辑：  技术翻译与对比。
矩阵扩展：
【翻译维】场景映射（参数 -> 体验）。
【博弈维】田忌赛马（强调碾压优势）。
【验证维】缺陷防御（主动披露非致命缺点）。
框架三：个性化生活方式型
适用：  美妆、旅游、装修。
核心逻辑：  痛点与解决方案绑定。
矩阵扩展：
【情境维】痛点归因（问题 -> 成分）。
【方案维】公式植入（问题 + 场景 = 产品）。
【结果维】周期反馈（真人实测数据）。
框架四：本地服务与体验型
适用：  餐饮、医美、健身。
核心逻辑：  LBS与特色提取。
矩阵扩展：
【氛围维】场景打标（私密、商务）。
【设施维】Schema 标记（营业时间、包间）。
【口碑维】评论摘要（引导提取高频好评词）。

第五部分：Query 知识图谱展示与输出
核心价值：为什么需要知识图谱？
传统的 Excel 关键词表是线性的，无法体现用户意图的关联性。GEO 系统通过构建 “品牌-场景-问题-答案” 的知识图谱，实现以下价值：
语义覆盖可视化：  直观看到品牌在“安全”这个话题下，是否覆盖了所有死角（如：是否缺失了“特殊人群”维度的内容？）。
RAG 结构化输入：  图谱数据可直接导入企业级 AI 搜索（RAG）系统，大幅提升 AI 回答的准确率。
动态热度监控：  结合搜索数据，用节点大小/颜色代表用户关注度，指导内容迭代。
图谱数据模型
系统基于 Property Graph (属性图) 模型构建，定义了 4 层节点结构：
🔴 根节点 (Root Node)
类型：  Product
示例：  问界M9
属性：  Brand, USP (华为智驾)
🟠 一级节点 (Seed Node)
类型：  Seed_Keyword
示例：  安全
关系：  (Product) -[HAS_TOPIC]-> (Seed_Keyword)
🟡 二级节点 (Dimension Node)
类型：  GEO_Dimension
示例：  主动能力维 , 风险合规维
关系：  (Seed_Keyword) -[EXPANDS_TO]-> (GEO_Dimension)
🟢 三级节点 (Query Node)
类型：  User_Query (即 Phase 3 产出的高仿真提问)
示例：  高速遇到静止车能刹住吗？
属性：
Angle : 切角 (如：极端场景)
Score : 仿真度评分 (95分)
Volume : 预测搜索热度 (High)
关系：  (GEO_Dimension) -[CAPTURES_INTENT]-> (User_Query)
6.3 可视化交互设计 (UI Specification)
前端 “场景图谱画布 (Graph Canvas)” 的交互规范：
力导向布局 (Force-Directed Layout)：
以产品为中心，向外辐射维度，最外层是密集的 Query 节点，形成“包围圈”。
颜色编码 (Color Coding)：
通过颜色区分 GEO 维度（如：红色代表风险维，蓝色代表技术维），帮助运营人员快速识别内容配比。
热力显示 (Heatmap Mode)：
节点大小：  代表该问题的搜索频次/重要性（基于 Phase 3 的 Demand Score）。
节点透明度：  代表内容填充状态（实心=已生成答案，半透明=待生成）。

点击 Query Node ，侧边栏弹出该问题的完整信息：
 
## 接口变更说明（/api/mine-queries）
- combineSeed 的品类参数改为由大模型基于品牌名实时分类（brandOnly），不再使用预置的 `getCategoryByBrand` 回退。
- 分类调用返回 JSON（仅包含 `category`），解析失败时继续生成但不附加品类前缀；响应体新增 `meta.brand` 与 `meta.category` 以便前端与监控核验。
- 错误处理与日志：分类失败不影响查询生成，监控后台可通过 operation=`decode_classify` 查看调用记录。
